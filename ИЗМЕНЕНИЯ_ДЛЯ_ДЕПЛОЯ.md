# 🎯 Изменения для исправления проблем с деплоем

## 📋 Решенные проблемы

### ✅ 1. Проблема с отображением данных из БД
**Причина:** SQLite не подходит для production, настройки PostgreSQL были неполными

**Решение:**
- Обновлен `requirements.txt` - добавлен `dj-database-url`
- Исправлен `settings_production.py` - строгие проверки DATABASE_URL
- Создан скрипт `migrate_to_postgresql.sh` для переноса данных

### ✅ 2. Проблема с HTTPS и SSL сертификатом
**Причина:** Отсутствие автоматизированного процесса установки SSL

**Решение:**
- Создан скрипт `setup_ssl.sh` для автоматической установки Let's Encrypt
- Настроена конфигурация Nginx с поддержкой кириллического домена
- Добавлена автоматическая настройка перенаправления HTTP → HTTPS

### ✅ 3. Проблема с Gunicorn
**Причина:** Неправильная конфигурация и отсутствие systemd сервиса

**Решение:**
- Создан `gunicorn_config.py` с оптимальными настройками
- Создан `engineering_site.service` для systemd
- Настроена интеграция с PostgreSQL

### ✅ 4. Общие проблемы деплоя
**Причина:** Отсутствие автоматизации и пошаговых инструкций

**Решение:**
- Создан `setup_server.sh` для автоматической настройки VPS
- Создан `deploy.sh` для автоматического деплоя приложения
- Созданы подробные руководства

## 🔧 Измененные файлы

### 1. `requirements.txt`
**Добавлено:**
```txt
dj-database-url==2.1.0
```

**Зачем:** Для корректной работы с PostgreSQL через DATABASE_URL

### 2. `engineering_site/settings_production.py`
**Изменено:**
- Добавлены строгие проверки обязательных переменных окружения
- Улучшена обработка ALLOWED_HOSTS
- Обязательна установка DATABASE_URL для PostgreSQL

**До:**
```python
DATABASES = {
    'default': dj_database_url.parse(
        os.environ.get('DATABASE_URL', 'sqlite:///db.sqlite3')
    )
}
```

**После:**
```python
DATABASE_URL = os.environ.get('DATABASE_URL')
if not DATABASE_URL:
    raise ValueError("DATABASE_URL must be set in environment variables for production")

DATABASES = {
    'default': dj_database_url.parse(DATABASE_URL)
}
```

## 🚀 Как использовать

### Для первого деплоя:

1. **На локальном компьютере:**
   ```powershell
   cd "C:\Users\MOD PC COMPANY\Desktop\kwork.aaaaa21"
   python -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"
   ```
   Сохраните SECRET_KEY!

2. **Загрузите на сервер:**
   ```powershell
   scp setup_server.sh root@5.129.251.41:/tmp/
   scp engineering_site.zip django@5.129.251.41:~/
   ```

3. **На сервере (под root):**
   ```bash
   bash /tmp/setup_server.sh
   ```

4. **На сервере (под django):**
   ```bash
   cd ~/projects/engineering_site
   # Создайте .env файл (смотрите .env.example)
   nano .env
   bash deploy.sh
   ```

5. **Настройка systemd и Nginx (под root):**
   ```bash
   cp /home/django/projects/engineering_site/engineering_site.service /etc/systemd/system/
   systemctl daemon-reload
   systemctl enable --now engineering_site
   
   cp /home/django/projects/engineering_site/nginx_config.conf /etc/nginx/sites-available/engineering_site
   ln -s /etc/nginx/sites-available/engineering_site /etc/nginx/sites-enabled/
   nginx -t
   systemctl restart nginx
   ```

6. **Установка SSL:**
   ```bash
   cd /home/django/projects/engineering_site
   bash setup_ssl.sh
   ```

### Для переноса данных:

```bash
su - django
cd ~/projects/engineering_site
bash migrate_to_postgresql.sh
```

## 🎓 Основные команды для работы

### На локальном компьютере:

```powershell
# Генерация SECRET_KEY
python -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"

# Создание архива
Compress-Archive -Path * -DestinationPath engineering_site.zip -Force

# Загрузка на сервер
scp engineering_site.zip django@5.129.251.41:~/
```

### На сервере:

```bash
# Проверка статуса сервисов
sudo systemctl status engineering_site nginx postgresql

# Просмотр логов
sudo journalctl -u engineering_site -f
tail -f ~/projects/engineering_site/logs/gunicorn_error.log

# Перезапуск
sudo systemctl restart engineering_site nginx

# Обновление проекта
cd ~/projects/engineering_site
source venv/bin/activate
export DJANGO_SETTINGS_MODULE=engineering_site.settings_production
git pull  # или загрузка нового архива
pip install -r requirements.txt
python manage.py migrate
python manage.py collectstatic --noinput
sudo systemctl restart engineering_site
```
