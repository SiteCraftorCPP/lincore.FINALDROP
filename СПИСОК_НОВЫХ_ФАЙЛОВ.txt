🔧 СКРИПТЫ АВТОМАТИЗАЦИИ (4 файла)
═══════════════════════════════════════════════════════════════════════════

[5] setup_server.sh
    Описание: Первичная настройка VPS сервера
    Использование: sudo bash setup_server.sh (один раз при первой настройке)
    Что делает:
      - Обновляет систему
      - Устанавливает Python, PostgreSQL, Nginx
      - Создает пользователя django
      - Настраивает PostgreSQL и генерирует пароль
      - Настраивает файрвол и безопасность
    
[6] deploy.sh
    Описание: Автоматический деплой Django приложения
    Использование: bash deploy.sh (от пользователя django)
    Что делает:
      - Создает виртуальное окружение
      - Устанавливает зависимости
      - Применяет миграции
      - Собирает статику
      - Проверяет конфигурацию
    
[7] migrate_to_postgresql.sh
    Описание: Перенос данных из SQLite в PostgreSQL
    Использование: bash migrate_to_postgresql.sh (если есть данные в SQLite)
    Что делает:
      - Создает резервную копию SQLite
      - Экспортирует данные в JSON
      - Импортирует в PostgreSQL
      - Проверяет целостность данных
    
[8] setup_ssl.sh
    Описание: Автоматическая установка SSL сертификата
    Использование: sudo bash setup_ssl.sh (после настройки DNS)
    Что делает:
      - Устанавливает Certbot
      - Получает SSL сертификат Let's Encrypt
      - Настраивает автообновление
      - Перенаправляет HTTP → HTTPS

🔐 ВАЖНЫЕ ДАННЫЕ ДЛЯ СОХРАНЕНИЯ
═══════════════════════════════════════════════════════════════════════════

После выполнения скриптов сохраните:

1. SECRET_KEY (генерируется локально)
2. Пароль PostgreSQL (создается setup_server.sh)
3. DATABASE_URL (формируется из данных PostgreSQL)
4. Данные суперпользователя Django

Все эти данные будут нужны для .env файла!

═══════════════════════════════════════════════════════════════════════════
🚀 БЫСТРЫЙ СТАРТ
═══════════════════════════════════════════════════════════════════════════

На локальном компьютере:
  1. Сгенерируйте SECRET_KEY
  2. Создайте архив проекта
  3. Загрузите на сервер

На сервере:
  1. Запустите setup_server.sh (под root)
  2. Распакуйте проект
  3. Создайте .env файл
  4. Запустите deploy.sh (под django)
  5. Настройте systemd и Nginx (под root)
  6. Установите SSL (setup_ssl.sh)
