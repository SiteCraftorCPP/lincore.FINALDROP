# Engineering Site - Инженерно-сервисная компания

Веб-сайт для инженерно-сервисной компании, специализирующейся на комплексном обслуживании инженерных систем.

## 🚀 Особенности

- **Адаптивный дизайн** - работает на всех устройствах
- **Многостраничный сайт** с различными услугами
- **Админ-панель Django** для управления контентом
- **Система заявок** с отправкой в админку
- **Галерея фотографий** с возможностью загрузки через админку
- **SEO-оптимизация** и быстрая загрузка

## 📋 Страницы сайта

1. **Главная страница** - общая информация о компании
2. **Комплексное обслуживание инженерных систем** - основные услуги
3. **ИТП и ЦТП «под ключ»** - тепловые пункты
4. **Поверка, ремонт и восстановление приборов учета** - поверка счетчиков
5. **Аварийная служба** - экстренные вызовы
6. **Монтаж инженерных коммуникаций** - установка систем
7. **Аудит инженерных систем для госзаказчиков** - аудит для госзаказчиков
8. **Подготовка к отопительному сезону** - сезонные работы
9. **Профессиональное техническое обслуживание** - климатические системы

## 🛠 Технологии

- **Backend:** Django 3.2.25
- **Frontend:** HTML5, CSS3, JavaScript
- **База данных:** SQLite (разработка), PostgreSQL (продакшен)
- **Сервер:** Gunicorn + Nginx
- **Статические файлы:** WhiteNoise

## 📦 Установка и запуск

### Локальная разработка

1. **Клонируйте репозиторий:**
```bash
git clone <repository-url>
cd engineering_site
```

2. **Создайте виртуальное окружение:**
```bash
python -m venv venv
# Windows
venv\Scripts\activate
# Linux/Mac
source venv/bin/activate
```

3. **Установите зависимости:**
```bash
pip install -r requirements.txt
```

4. **Примените миграции:**
```bash
python manage.py migrate
```

5. **Создайте суперпользователя:**
```bash
python manage.py createsuperuser
```

6. **Запустите сервер:**
```bash
python manage.py runserver
```

Сайт будет доступен по адресу: `http://127.0.0.1:8000/`

## 🔧 Настройка

### Переменные окружения

Создайте файл `.env` в корне проекта:

```env
SECRET_KEY=your-secret-key-here
DEBUG=True
ALLOWED_HOSTS=localhost,127.0.0.1
DATABASE_URL=sqlite:///db.sqlite3
```

### База данных

По умолчанию используется SQLite. Для продакшена рекомендуется PostgreSQL:

```env
DATABASE_URL=postgresql://user:password@localhost:5432/dbname
```
