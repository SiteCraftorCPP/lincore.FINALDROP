# 🚀 Инструкции по деплою на Git

## Подготовка проекта к загрузке на GitHub

### 1. Проверка статуса Git

```bash
git status
```

### 2. Добавление всех изменений

```bash
# Добавить все измененные файлы
git add .

# Или добавить конкретные файлы
git add website/models.py
git add website/admin.py
git add website/views.py
git add website/templates/website/audit.html
git add static/css/styles.css
git add static/js/main.js
```

### 3. Создание коммита

```bash
git commit -m "feat: Добавлена модель CommercialProposal и админ-интерфейс

- Создана модель CommercialProposal для управления КП
- Добавлен админ-интерфейс для загрузки изображений КП
- Обновлен шаблон audit.html для динамического контента
- Исправлены кнопки навигации на всех страницах
- Добавлена кнопка 'Заказать звонок' в шапку
- Исправлены проблемы с индикаторами прогресса
- Обновлены стили для мобильной версии"
```

### 4. Загрузка на GitHub

```bash
# Если репозиторий уже существует
git push origin main

# Если создаете новый репозиторий
git remote add origin https://github.com/username/repository-name.git
git branch -M main
git push -u origin main
```

## 🔧 Дополнительные настройки

### Создание .env файла для продакшена

```env
SECRET_KEY=your-production-secret-key
DEBUG=False
ALLOWED_HOSTS=yourdomain.com,www.yourdomain.com
DATABASE_URL=postgresql://user:password@localhost:5432/dbname
SECURE_SSL_REDIRECT=True
```

### Настройка для VPS деплоя

1. **Скопируйте файлы на сервер:**
```bash
scp -r . user@server:/path/to/project/
```

2. **Запустите скрипт настройки:**
```bash
bash setup_server.sh
```

3. **Запустите деплой:**
```bash
bash deploy.sh
```

## 📝 Важные файлы для деплоя

### Обязательные файлы:
- `requirements.txt` - зависимости Python
- `manage.py` - управление Django
- `engineering_site/settings_production.py` - настройки продакшена
- `deploy.sh` - скрипт деплоя
- `setup_server.sh` - настройка сервера
- `nginx_config.conf` - конфигурация Nginx
- `engineering_site.service` - systemd сервис

### Медиа файлы:
- `media/` - загружаемые пользователями файлы
- `static/` - статические файлы (CSS, JS, изображения)

## 🚨 Важные моменты

1. **Не коммитьте:**
   - `.env` файлы с секретными ключами
   - `db.sqlite3` (база данных)
   - `__pycache__/` папки
   - Медиа файлы (если они большие)

2. **Обязательно добавьте:**
   - Все исходные файлы проекта
   - Конфигурационные файлы
   - Скрипты деплоя
   - Документацию

3. **Проверьте перед коммитом:**
   - Все файлы добавлены в `.gitignore`
   - Нет секретных данных в коде
   - Все зависимости указаны в `requirements.txt`
